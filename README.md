# SmartGroupClashes (Navisworks)

Плагин для **Autodesk Navisworks Manage**: группировка результатов тестов коллизий по выбранным правилам. Рабочее имя продукта и сборки — **SmartGroupClashes**; нумерация версий с **1.0.0** (отдельно от истории исходного GroupClashes).

## Исходный проект

Этот репозиторий развивается **на основе** открытого плагина **Simon Moreau**; текущая ветка и сборки — **Kanukhin_A** / **SmartGroupClashes**:

**Исходная программа на GitHub:** [https://github.com/simonmoreau/GroupClashes](https://github.com/simonmoreau/GroupClashes)

## Отличия от исходного кода

Ниже — основные изменения относительно типичной версии из репозитория Simon Moreau (ветка/снимок на момент форка могут отличаться).

### Сборка и поставка

- Добавлены конфигурации **Navisworks 2026** (`2026Debug` / `2026Release`) и запись в `PackageContents.xml` для этой версии.
- Атрибуты плагина Navisworks и bundle используют идентификатор **`SmartGroupClashes`** (вместо старого `GroupClashes`); автор пакета и изготовитель MSI — **`Kanukhin_A`**; обновлён `FindPlugin` для панели.
- **`PostBuild.ps1`**: копирование в локальный каталог плагинов Navisworks, без подписи кода; опциональная упаковка на внешний диск и в ZIP при наличии пути и `7z`.
- В каталоге **`Installer/`** — сценарий сборки **MSI** через WiX (`dotnet tool install wix`); для пользователя удобнее запускать **`Build-Msi.bat`** в корне репозитория (см. [«Сборка MSI»](#сборка-msi)).

### Интерфейс

- Интерфейс панели **на русском** (подписи, кнопки, подсказки, строки прогресса и транзакций в коде).
- Увеличены размеры окна, шрифт **Arial**, поле **поиска** по списку тестов с подсказкой-плейсхолдером.
- Поддержка **мультивыбора** тестов коллизий для группировки / разгруппировки сразу в нескольких тестах.
- Выпадающие списки режимов показывают **русские названия** режимов (через `GroupingModeOption`).
- Исправлено поведение **WinForms `ElementHost` + WPF**: панель нормально подстраивается под размер окна Navisworks (без жёсткой «залипшей» ширины из-за `AutoSize` и лишних фиксированных размеров).

### Логика группировки

- **Пересечение осей**: группировка по осям **без привязки к этажу** (ключ группы без суффикса уровня из `DisplayName`, чтобы коллизии на разных уровнях с теми же осями попадали в одну группу при выборе только режима по осям).
- Новый режим **«Имя элемента»**: группировка по отображаемым именам участвующих элементов.
- Режимы **«Имя семейства»** и **«Имя типа»**: группировка по свойствам модели (как в отчётах Revit/NWC), с поиском полей с подписями на русском и английском.
- Опция **не создавать группы**, в которых **все** пересечения уже в статусах «исправлено» (логика `Resolved` / `Approved` в API).
- Для групп по **статусу** в названии группы выводятся **русские подписи** известных статусов Navisworks (остальные — как в API).

### Производительность

- Меньше повторных вызовов **`ClosestIntersection`** в группировке по уровню и по сетке.
- Корректная сортировка для сетки: **`OrderBy` + `ThenBy`** вместо цепочки двух `OrderBy`.
- Более дешёвое удаление групп из списков (обход с конца и `RemoveAt`).

### Прочее

- Локализация ленты на русском; в `PackageContents.xml` имя пакета — **SmartGroupClashes**, отображаемые строки интерфейса по-прежнему на русском.
- Удалены артефакты с привязкой к старому имени сборки из кэша сборки при необходимости (после смены имени DLL рекомендуется **чистая пересборка**).

## Сборка

> В репозитории заранее скомпилированные `SmartGroupClashes.dll` доступны только для части версий Navisworks (только для тех конфигураций, которые уже были собраны и лежат в `GroupClashes\bin\*Release`).  
> Если нужна другая версия Navisworks, пересоберите проект под соответствующую конфигурацию года в Visual Studio.

1. Откройте **`GroupClashes.sln`** в Visual Studio.
2. Выберите конфигурацию года Navisworks, например **`2026Release`** (или другой доступный `*Release` / `*Debug`).
3. Соберите проект **SmartGroupClashes**.

Результат: **`GroupClashes\bin\<конфигурация>\SmartGroupClashes.dll`** и сопутствующие файлы (`en-US`, иконки и т.д.).

После сборки **`PostBuild.ps1`** копирует bundle в:

`%APPDATA%\Autodesk\ApplicationPlugins\SmartGroupClashes.bundle\`

## Сборка MSI

Сборка установщика выполняется **`Build-Msi.bat`** в корне репозитория (двойной щелчок или ярлык — без ручного запуска PowerShell). Батник вызывает внутренний сценарий **`Installer\Build-Msi.ps1`** с политикой **`Bypass`**. В один **MSI** попадают все версии плагина, которые вы заранее собрали в конфигурациях **`2024Release`**, **`2026Release`** и т.д.: в каждой такой папке `GroupClashes\bin\` должен лежать **`SmartGroupClashes.dll`** и сопутствующие файлы (как после сборки в Visual Studio).

### Полноценный установщик и состав MSI

Это **обычный продуктовый MSI Windows Installer**, а не «заглушка» или ссылка на интернет:

- В каталог **`Installer\staging\SmartGroupClashes.bundle\`** копируется **всё содержимое** каждой собранной папки **`bin\<год>Release\`** (рекурсивно): скомпилированная **`SmartGroupClashes.dll`**, **`en-US`** (`.name`, `.xaml`), **`Images`** (иконки) и любые другие файлы вывода сборки.
- WiX упаковывает эти файлы в MSI с **встроенным CAB** (`MediaTemplate EmbedCab="yes"`): на целевой машине **ничего докачивать не нужно** — при установке файлы распаковываются в **`%AppData%\Autodesk\ApplicationPlugins\SmartGroupClashes.bundle\`**.
- Перед сборкой MSI скрипт **проверяет**, что для **каждого** включённого года в bundle есть **`SmartGroupClashes.dll`** ненулевого размера; иначе сборка прервётся с ошибкой (чтобы не получить пустой установщик).
- Устаревшие артефакты **`GroupClashes.dll`** и старые **`GroupClashesIcon*.ico`** из staging **удаляются**, чтобы в установщик не попала чужая сборка рядом с актуальной.
- Версия для MSI приводится к виду **`a.b.c.d`** (четыре числа 0…255), как ожидает Windows Installer; имя файла MSI использует эту же строку (например, **`SmartGroupClashes-1.0.0.0.msi`** при `AppVersion="1.0.0"` в `PackageContents.xml`).

### Что понадобится

На компьютере должны быть установлены **.NET SDK** (команда `dotnet --version` в консоли должна работать) и **PowerShell** (им пользуется батник под капотом). Отдельно устанавливать WiX не требуется: при первом запуске сценарий создаёт в корне репозитория локальный манифест инструментов и подтягивает пакет **WiX** через `dotnet tool install wix`; для этого один раз нужен доступ в интернет.

### Шаг 1. Собрать плагин для всех нужных версий Navisworks

Переключайтесь в Visual Studio по очереди на каждую конфигурацию **«год + Release»** и выполняйте сборку проекта **SmartGroupClashes**. Повторите для каждой версии Navisworks, которую хотите положить в один MSI.

Альтернатива — командная строка (подставьте свой путь к репозиторию):

```powershell
cd C:\путь\к\GroupClashes-master
dotnet msbuild "GroupClashes.sln" /t:Build /p:Configuration=2024Release /p:Platform="Any CPU"
dotnet msbuild "GroupClashes.sln" /t:Build /p:Configuration=2026Release /p:Platform="Any CPU"
```

В каталоге **`GroupClashes\bin\<год>Release\`** после сборки должны появиться **`SmartGroupClashes.dll`**, папки **`en-US`**, **`Images`** и остальное, что копируется в bundle.

Скрипт MSI учитывает **только** каталоги с именем **четыре цифры + `Release`** (например `2024Release`). Сборки в конфигурациях **`*Debug`** в установщик **не** попадают.

### Шаг 2. Сформировать MSI

После шага 1 откройте **корень репозитория** (рядом с `GroupClashes.sln` лежит **`Build-Msi.bat`**) и запустите **`Build-Msi.bat`**:

- **Двойной щелчок** в Проводнике — откроется окно консоли и выполнится сборка MSI (политика выполнения скриптов не мешает: её задаёт сам батник).
- Либо сделайте **ярлык** на `Build-Msi.bat` — это удобная «одна кнопка» после сборки DLL в Visual Studio.

Батник подготовит каталог **`Installer\staging\SmartGroupClashes.bundle\`**: скопирует **`PackageContents.xml`** и содержимое каждой найденной папки **`bin\<год>Release\`** в **`Contents\<год>\`**, удалит **`.pdb`**, сгенерирует **`Installer\Product.generated.wxs`** и вызовет WiX. Готовый установщик по умолчанию: **`Installer\artifacts\SmartGroupClashes-<версия>.msi`**, где `<версия>` берётся из **`AppVersion`** в `PackageContents.xml` или из параметров ниже и **нормализуется до четырёх чисел** для MSI.

В консоли перед WiX выводится сводка: сколько файлов попадёт в MSI и список **`.dll`** с размерами; затем — путь к MSI и список включённых конфигураций годов.

### Дополнительные параметры (через батник)

Все параметры ниже **необязательны**; значения по умолчанию заданы во **`Installer\Build-Msi.ps1`** (батник просто передаёт их дальше после `%*`).

| Параметр | Для чего используется |
|----------|------------------------|
| `-PluginProjectDir` | Папка с проектом плагина и файлом `PackageContents.xml` (если структура репозитория нестандартная). |
| `-OutputDir` | Куда сохранить готовый MSI (по умолчанию — `Installer\artifacts`). |
| `-Version` | Номер версии в свойствах пакета и в имени файла (по умолчанию — из `AppVersion` в `PackageContents.xml`; для MSI дополняется до **четырёх** компонентов, например `1.0.0` → `1.0.0.0`). |
| `-Manufacturer` | Строка «Изготовитель» в свойствах установщика. |
| `-ProductName` | Название продукта в мастере установки и в списке программ. |
| `-UpgradeCode` | Постоянный идентификатор продукта в формате GUID. Его **не следует** менять без нужды: иначе Windows будет считать новую сборку другим приложением, а не обновлением. |

Пример: другая версия и свой каталог для MSI (из корня репозитория, в адресной строке Проводника или в «Выполнить»):

```bat
Build-Msi.bat -Version "1.2.0" -OutputDir "C:\Сборки\Msi"
```

### Что делает установка MSI

Установщик рассчитан на **текущего пользователя Windows** (не на «всех пользователей компьютера»): файлы попадают в каталог

**`%AppData%\Autodesk\ApplicationPlugins\SmartGroupClashes.bundle\`**

Внутри одного MSI лежат все годы Navisworks, которые вы включили на шаге 1; при запуске Navisworks подхватит нужную вложенную папку **`Contents\<год>\`** согласно правилам в **`PackageContents.xml`**.

Если **`Build-Msi.bat`** завершится с ошибкой, в консоли будет сообщение на русском языке с пояснением (чаще всего — не найден ни один каталог `*Release` с `SmartGroupClashes.dll`: в этом случае вернитесь к шагу 1).

## Миграция со старой установки

Если ранее стоял пакет с другим именем папки (например, со старым суффиксом в имени bundle), удалите старую папку из:

`%APPDATA%\Autodesk\ApplicationPlugins\`

чтобы не было **двух копий** плагина одновременно.
